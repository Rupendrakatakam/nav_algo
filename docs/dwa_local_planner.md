# DWA Local Planner

## Overview
The Dynamic Window Approach (DWA) is a local reactive obstacle-avoidance algorithm that computes the optimal velocity command $(v, \omega)$ for the robot to execute in the next time frame, typically avoiding unmapped, dynamic obstacles while tracking the `global_path` carrot from the A* planner.

## How it Works
At its core, DWA is a forward-simulation evaluation approach:
1. **Dynamic Window Generation:** We sample a defined subset of valid linear velocities ($v$) and angular velocities ($\omega$) based on the robot's kinematic limits.
2. **Trajectory Rollout:** For each velocity pair $(v, \omega)$, we predict the kinematic trajectory of the robot over a short lookahead simulation time (e.g. 2.0 seconds) assuming it holds those velocities.
3. **Cost Evaluation:** We score the predicted trajectory's final state based on three core behavioral objectives. 

### Objective Function
We try to find the commands that yield the lowest total cost. The cost equation evaluates the endpoints of these projected trajectories:
$$C = \alpha \cdot \text{dist\_to\_goal} + \beta \cdot \text{max\_obstacle\_cost} + \gamma \cdot \text{heading} - \delta \cdot \text{speed}$$

* **Distance to Goal / Path (`dist_to_goal`):** Encourages moving toward the current pure pursuit lookahead carrot (generated by the global planner).
* **Obstacle Clearance (`max_obstacle_cost`):** The single most critical term. We scan the entire 2.0 second trajectory. If any $(x, y)$ coordinate lands inside an obstacle or extremely close to one ($cost = 100$), the command is immediately aborted as `crashed = true`. For near-miss trajectories, the peak cost from the `costmap` is extracted and heavily penalized.
* **Heading Alignment (`heading`):** Penalizes trajectories that point the robot away from the optimal angle pointing toward the carrot.
* **Velocity Reward (`speed`):** We subtract the final velocity to reward the robot for maintaining its top speed where safe to do so. 

## Dynamic Avoidance and "The Fear Factor"
An aggressive velocity reward can occasionally cause the robot to drive dangerously close to an approaching dynamic obstacle, leading to circumstances where it becomes completely enveloped in the obstacle's lethal radius. If a robot begins an evaluation frame *inside* an obstacle, all predicted trajectories instantly crash, leading to a permanent freeze.

To resolve this, we impose an overwhelming obstacle penalty factor (`100.0` weight). This extreme fear coefficient ensures any near-miss trajectories are instantly out-scored by evasive detours, or by stopping entirely before entering the narrow gap. The robot safely yields or swerves widely, remaining outside the inflated collision boundary.
